# PRD - Google AI 업무 도구 분류 맵 (인터랙티브 관계도)

> 문서 목적: `google-ai-tools-chord-visualization.html` 산출물을 처음부터 다시 만들 수 있도록 요구사항을 정의한다.
> 산출물 유형: 단일 HTML 파일 (Vanilla JS + 인라인 SVG, 빌드 도구 없음)
> 용도: 패스트캠퍼스 '일잘러를 위한 구글 AI 생태계 바이블' 강의 보조 인터랙티브 자료
> 최종 수정: 2026-06-24

---

## 1. 배경 & 문제 정의

구글 AI 생태계는 Gemini, Workspace AI, 이미지·영상 생성, 노코드 앱, 에이전트 개발까지 **30여 개 도구**로 빠르게 팽창했다. 학습자는 다음 문제를 겪는다.

- 도구가 너무 많아 **무엇부터 써야 할지** 막막하다.
- 비슷해 보이는 도구(NotebookLM vs Gem, Opal vs Workspace Studio)의 **차이가 안 보인다.**
- 도구를 **단순 나열한 표**로는 관계와 진입 난이도가 전달되지 않는다.

→ 30여 개 도구를 **3개 분류축**으로 한 화면에서 탐색·비교하게 만드는 인터랙티브 관계도가 필요하다.

## 2. 목표 (Goals)

| 구분 | 내용 |
|------|------|
| G1 | 30여 개 구글 AI 도구를 한 화면(1080p 기준)에서 조망 |
| G2 | 분야 / 사용 난이도 / 활용 방식 3개 기준으로 즉시 그룹핑·비교 |
| G3 | 클릭 한 번으로 "이 묶음에 무엇이 있고, 어떻게 다른가"를 비교표로 확인 |
| G4 | 강의 랜딩으로 자연스럽게 연결되는 CTA 배너 노출 |
| G5 | 별도 설치·로그인·백엔드 없이 파일 하나로 동작 (강의/배포 어디서나 열림) |

### Non-Goals (Out of Scope)
- 실데이터 연동/검색 API, 로그인, 즐겨찾기 저장
- 도구별 상세 튜토리얼(설명은 1~2문장 요약 수준)
- 모바일 우선 설계 (반응형은 fallback 수준, 데스크톱 가로 화면 1순위)
- 다국어 (한국어 단일)

## 3. 타깃 사용자

- **1차**: 강의 수강생 / 직장인 일잘러 - 구글 AI 도구의 큰 그림을 잡고 싶은 비개발자
- **2차**: 강사(발표 화면용), 콘텐츠 소재(뉴스레터·블로그 캡처)로 재활용하는 운영자

## 4. 핵심 사용자 시나리오

1. 화면을 연다 → 중앙에 원형으로 배치된 도구 노드와, 우상단 분류칩 박스가 보인다. (온보딩 포인터 큐가 클릭 위치를 가리킨다)
2. 우상단 **'분야 > Workspace' 칩**을 클릭 → 해당 도구들이 공통 허브로 연결되고, 나머지는 흐려진다. 우측 패널이 **Workspace 비교표**로 바뀐다.
3. 비교표에서 각 도구의 '구분 포인트'를 읽고 차이를 이해한다.
4. 관심 도구 **노드를 직접 클릭** → 우측 패널이 대표 이미지 + 설명이 담긴 **상세 카드**로 바뀐다.
5. 이미지를 클릭 → **라이트박스**로 확대해서 본다.
6. 하단/상단 **강의 배너 CTA** 클릭 → 패스트캠퍼스 강의 랜딩으로 이동.

## 5. 정보 구조 & 데이터 모델

### 5.1 도구(Tool) 스키마
각 도구는 아래 속성을 가진 객체이며, JS 배열 `tools[]`에 정의한다.

```js
{
  name:     String,            // 표시명 (예: 'Gemini')
  icon:     String,            // 노드 안 글리프 (예: '✦', 'NB', '</>')
  domain:   String,            // 분야 (6종 중 1)
  function: String,            // 사용 난이도 (4종 중 1)
  usage:    String | String[], // 활용 방식 (10종 중 1~2). 배열 허용 (예: gcloud CLI)
  desc:     String,            // 1~2문장 요약 설명 + '구분 포인트'
  id:       Number             // 렌더 시 자동 부여 (index)
}
```

### 5.2 분류축 ① 분야 (domain) - 6종
표시 라벨 `분야` / 코드 `FIELD`. 색상은 구글 브랜드 팔레트 매핑.

| 분야 | 색상 | 소속 도구 |
|------|------|-----------|
| Google AI | `#4285F4` | Gemini, Gem, Gemini Live, Google AI Studio, AI Mode, Gemini in Chrome |
| 리서치·학습 | `#0097C2` | Deep Research, NotebookLM |
| Workspace | `#188038` | Gmail AI, Doc AI, Sheets AI, Slides AI, Meet AI, Forms AI, Workspace Studio, Workspace CLI, Vids |
| 이미지·영상·콘텐츠 | `#FBBC04` | Nano Banana, Flow, Pomeli, Mixboard, Text FX |
| 앱·서비스 | `#EA4335` | Disco, AppSheet, Opal, Firebase, Stitch |
| 개발·자동화 | `#A142F4` | Apps Script, Antigravity, ADK, gcloud CLI, Gemma |

`domainOrder` 배열이 원형 배치 순서를 결정한다: `['Google AI', '리서치·학습', 'Workspace', '이미지·영상·콘텐츠', '앱·서비스', '개발·자동화']`

### 5.3 분류축 ② 사용 난이도 (function) - 4종
표시 라벨 `사용 난이도` / 코드 `LEVEL`. 칩에 그라데이션(쉬움→어려움) 적용.

| 단계 | 그라데이션 (start→end) | 정의 |
|------|------------------------|------|
| 바로 사용 | `#4285F4`→`#34A853` | 설치/구조화 없이 계정 안에서 바로 사용 |
| 설정 필요 | `#34A853`→`#FBBC04` | 자료 업로드·연결·조건 지정 등 1회 세팅 필요 |
| 학습 필요 | `#FBBC04`→`#F29900` | 사용법·프롬프트·편집 기준 학습 필요 |
| 훈련 필요 | `#F29900`→`#EA4335` | 에이전트·모델·워크플로우를 길들여 사용 |

`levelOrder`로 칩 노출 순서 고정. (난이도 오름차순)

### 5.4 분류축 ③ 활용 방식 (usage) - 10종
표시 라벨 `활용 방식` / 코드 `USE`. `usageOrder`로 순서 고정, `usageToolOrder`로 비교표 내 도구 정렬 순서 지정.

`일상 업무 보조 / 자료 기반 생성 / 자료 조사·아이디어 기획 / 문서·보고서·데이터 / 회의·커뮤니케이션 정리 / 제안서·발표 준비 / 홍보·마케팅 / 반복 업무 자동화 / 업무용 도구 만들기 / 에이전트 자동화`

### 5.5 그룹 설명 (groupDescriptions)
20개 그룹(6+4+10) 각각에 `[이모지, 표시명, 한 줄 설명]`을 매핑. 그룹 활성화 시 상단 '그룹 노트'에 노출.

## 6. 기능 요구사항 (FR)

### FR-1. 방사형 관계도 (Radial Map) - SVG
- `viewBox="0 0 960 860"`, 중심 `cx=480, cy=470`, 도구 노드 배치 반지름 `r=284`.
- **배경 그리드**: 동심원 5개(반지름 `[132,220,302,356,392]`) + 64개 방사선(ray).
- **도메인 아크**: 분야별로 노드를 감싸는 호(arc)를 분야 색으로 그리고, 바깥쪽에 라벨 칩(배경+점+텍스트) 배치.
- **노드**: 각 도구를 원(ring+core)으로 그리고 중앙에 아이콘 글리프, 바깥에 회전 정렬된 라벨. 분야 색을 `--domain-color`로 상속.
- 각도 계산: 상단(-90°) 시작, 약간의 여백(0.09rad)을 두고 전체 원주에 균등 분배(`angleFor(i)`).

### FR-2. 분류칩 박스 (Taxonomy) - 우상단
- 3개 컬럼(분야 / 사용 난이도 / 활용 방식), 각 컬럼 헤더에 라벨+코드.
- 칩 클릭 시 `activateGroup(type, name)` 호출.
- 유휴 상태에서 `attract` 글로우 애니메이션으로 클릭 유도. 활성화되면 해제.
- 분야 칩은 색 점(dot), 난이도 칩은 그라데이션 배경.

### FR-3. 그룹 활성화 인터랙션 (핵심)
분류칩 클릭 시:
- 해당 그룹 소속 도구만 **공통 허브(hub)로 스포크(곡선 엣지) 연결**. 허브 위치는 소속 도구 각도의 원형 평균, 반지름은 type별(domain 174 / function 122 / usage 86).
- 소속 노드는 `hot`(강조), 비소속은 `dimmed`(흐림).
- 선택 칩은 `hot`, 상단 그룹 노트는 해당 설명으로 갱신(난이도는 기본 노트 유지).
- 엣지는 2개 이상일 때만 그린다(단일 도구 그룹은 스포크 생략).

### FR-4. 비교표 패널 (Comparison Table) - 우측
- 분류칩 클릭 시 우측 패널이 `[툴 | 분야 | 사용 난이도 | 활용 방식 | 구분 포인트]` 표로 전환.
- 제목/설명/도구 수 동적 갱신. usage 그룹은 `usageToolOrder` 순서로 정렬.
- 각 행에 도구 썸네일 포함(있을 경우).

### FR-5. 도구 상세 (Tool Detail)
- 노드 직접 클릭 시 우측 패널이 대표 이미지(hero) + 도구명 + 설명 + 태그(분야/난이도/활용) 카드로 전환.
- 칩 선택 상태는 해제, 클릭한 노드만 강조.

### FR-6. 이미지 라이트박스
- 썸네일/히어로 이미지 클릭 시 전체 화면 확대(backdrop blur). 닫기 버튼 / 배경 클릭 / ESC로 닫힘.

### FR-7. 그룹 노트 (상단 안내)
- 관계도 상단 중앙에 현재 선택 상태 안내(이모지+제목+설명). 미선택 시 기본 안내 문구.

### FR-8. 온보딩 포인터 큐
- '분류칩 클릭' / '툴 원 클릭'을 가리키는 떠다니는 포인터 2개. 진입 시 클릭 유도.

### FR-9. 강의 CTA 배너
- 헤더 영역에 강의 썸네일 + 제목 + 설명 + '강의 자세히 보기' 버튼.
- 링크: 패스트캠퍼스 구글 AI 생태계 바이블 (utm 파라미터 포함).
- 이미지 비율 보존(`object-fit: cover`), 외부 호스팅 이미지는 `loading="lazy"`.

### FR-10. 반응형
- `max-width: 1200px`에서 1컬럼 스택 레이아웃으로 전환, body 스크롤 허용, 관계도 고정 높이(780px).

## 7. 디자인 요구사항

- **무드**: 아이보리(`#f6f1e7`) 배경 + 글래스모피즘 패널 + 구글 6색 액센트. 밝고 정돈된 '제품 다이어그램' 톤.
- **배경**: radial-gradient 3겹(구글 색) + linear-gradient + SVG noise 오버레이(soft-light).
- **타이포그래피**: `Inter`(영문/UI), `Noto Sans KR`(한글), `JetBrains Mono`(코드/라벨). Google Fonts 로드.
- **컬러 토큰**: `:root` CSS 변수로 텍스트/보더/구글 6색 관리. `color-mix()`로 칩 그라데이션 동적 생성.
- **모션**: glow/float/attract 등 미세 애니메이션. **`prefers-reduced-motion: reduce` 시 전부 비활성화** (접근성).
- **워터마크**: 우하단 시그니처 텍스트.

## 8. 기술 스택 & 구현 제약

| 항목 | 결정 |
|------|------|
| 렌더링 | 인라인 SVG (DOM 직접 생성, `document.createElementNS`) |
| 로직 | Vanilla JS (프레임워크/번들러 없음) |
| 스타일 | 단일 `<style>` 블록, CSS 변수 + `color-mix` |
| 외부 의존성 | Google Fonts(CDN), 도구 이미지(`google-ai-tools-chord-visualization-assets/*.png`), 배너 이미지(원격 URL) |
| 파일 | 단일 `.html` 1개로 자기완결(이미지 폴더 동반) |
| 데이터 추가 | `tools[]` 배열에 객체 push → 자동 배치/렌더 (코드 수정 불필요) |

### 이미지 매핑
- `imageForTool(t)`가 도구명 → `*-assets/파일명.png` 매핑. 매핑 없으면 썸네일 미표시(graceful).
- 신규 도구 추가 시 동일 규칙으로 PNG를 assets 폴더에 추가.

## 9. 접근성 (A11y)
- SVG에 `role="img"`, 섹션 `aria-label` 지정.
- 라이트박스 `aria-hidden` 토글, ESC 닫기.
- 모션 민감 사용자 대응(`prefers-reduced-motion`).
- (개선 여지) 칩/노드 키보드 포커스·탭 순서, 색상 외 구분 보강 → 향후 과제.

## 10. 비기능 요구사항
- **성능**: 노드 32개 + 엣지 수십 개 규모, 정적 1회 렌더. 추가 최적화 불필요.
- **호환성**: 최신 Chrome/Edge/Safari (CSS `color-mix`, `backdrop-filter` 사용 - 구형 브라우저 미지원 감안).
- **오프라인**: 폰트/원격 배너 이미지 미로딩 시에도 핵심 관계도는 동작해야 함(폴백 폰트, lazy).

## 11. 데이터 정합성 규칙
- `tools[].domain`은 반드시 `domainColors` 키 중 하나.
- `tools[].function`은 `levelOrder` 중 하나.
- `tools[].usage`의 모든 값은 `usageOrder`에 존재해야 함(비교표 정렬 보장).
- 신규 그룹 추가 시 `groupDescriptions`에 설명 항목 동반 추가(없으면 기본 폴백 문구 사용).

## 12. 인수 기준 (Acceptance Criteria)
- [ ] 30여 개 도구가 원형으로 배치되고 각 노드에 아이콘·라벨이 보인다.
- [ ] 분야/난이도/활용 칩을 클릭하면 해당 도구만 허브로 연결되고 비교표가 갱신된다.
- [ ] 노드를 클릭하면 대표 이미지+설명 상세 카드가 뜬다.
- [ ] 이미지 클릭 시 라이트박스 확대, ESC/배경 클릭으로 닫힌다.
- [ ] CTA 배너가 강의 랜딩으로 정확히 연결된다(utm 포함).
- [ ] 1200px 이하에서 레이아웃이 깨지지 않고 스택된다.
- [ ] `prefers-reduced-motion`에서 애니메이션이 멈춘다.
- [ ] `tools[]`에 도구 1개를 추가하면 코드 수정 없이 배치/비교표/난이도 그룹에 반영된다.
